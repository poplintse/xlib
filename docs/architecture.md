# XLib 架构

本文记录当前实现的稳定边界。产品交互以
[`product-requirements.md`](product-requirements.md) 为准；本文只说明数据归属、阅读定位、缓存、目录、书签和运行状态如何协作。

## 1. 页面与状态边界

应用由单个 `MainActivity` 组织书架、阅读、搜索、目录/书签和设置页面。页面切换前必须显式处理阅读位置、自动翻页和临时搜索状态。

- 书架、设置、目录/书签、搜索和阅读共用全局浅色/深色主题。
- 正式阅读使用 `Book.offset` 作为主位置；搜索结果阅读使用 `SearchSession` 保存进入搜索前的位置，临时阅读不得覆盖正式进度。
- 目录或书签跳转会清除搜索临时阅读状态，更新 `Book.offset` 后重新进入正式阅读框架。
- 自动翻页属于纯运行时状态，不持久化。进入搜索、目录/书签、设置、书架、后台或离开阅读框架时立即关闭。

## 2. 持久化模型

### 2.1 书籍 metadata

`BookStore` 把书籍列表保存到 `SharedPreferences` 的 `books` JSON 数组。每本书包含：

- `id`、可编辑书名 `title`、导入时完整文件名 `sourceName`、作者 `author`；
- 本地副本路径 `path`、文件大小 `fileSize`、编码 `encoding`；
- 权威阅读位置 `offset`、派生显示进度 `progress`、最后阅读时间 `updatedAt`；
- 翻页模式等与单本书绑定的必要状态。

历史数据若只有百分比而没有 offset，加载时允许用 `fileSize * progress` 做一次迁移；迁移完成后仍以 offset 为权威。

普通翻页只在主线程更新内存中的 offset；连续翻页产生的 metadata 保存以约 `600 ms` 防抖合并。页面切换和 `onPause()` 仍执行即时保存，防抖不得造成退出时丢失进度。

### 2.2 全局设置

以下设置直接保存在同一个 `SharedPreferences` 中，不再写入单本书：

- 应用主题；
- TXT 自动生成目录；
- 阅读时锁屏；
- 自动翻页间隔；
- 触摸灵敏度；
- 字体、字号和行间距。

`ReaderSettingsOptions` 负责范围归一化，`ReaderDisplayPolicy` 负责把设置转换为触摸阈值、字体和行距等显示参数，`ReaderRuntimePolicy` 负责自动翻页和亮屏的运行条件。旧版单书主题、字号等字段不再是有效数据源。

| 偏好键 | 默认值与有效范围 | 实际消费方式 |
| --- | --- | --- |
| `app_theme` | 浅色；浅色/深色 | 所有页面重建时应用颜色令牌和系统栏颜色。旧值“跟随系统”只在启动迁移一次，按当时系统外观写为固定浅色或深色。 |
| `auto_toc` | 关闭 | 开启时安排缺失目录的后台扫描；关闭后待执行任务在读取和写入前再次检查开关，不再写入自动生成结果。 |
| `keep_screen_on` | 关闭 | 仅在正式阅读框架显示时添加 `FLAG_KEEP_SCREEN_ON`；离开阅读、进入后台或设置页时清除，返回阅读或恢复前台时按偏好恢复。 |
| `auto_page_interval` | 15 秒；10–30 秒，步长 1 秒 | 阅读页点击自动翻页按钮后换算为主线程延迟；自动翻页开关本身不保存。 |
| `sensitivity` | 中；高/中/低 | 分别影响点击容差 20/16/12dp 和滑动阈值 48/72/96dp。 |
| `font_family` | 系统；系统/黑体/宋体/仿宋/等宽 | 映射到正文 `Typeface`；当前“仿宋”使用系统 `serif` 的斜体近似，未内置字体文件。 |
| `font_size` | 20sp；14–34sp | 应用于正文 `setTextSize()`，随后重新适配完整文本行与当前阅读位置。 |
| `line_spacing` | 18%；10%–40% | 换算为额外行距后调用正文 `setLineSpacing()`。 |

### 2.3 目录与书签

- `TocStore` 将每本书的目录保存到应用内部 `files/toc/<bookId>.json`。文件记录源文件大小、修改时间、编码以及每个目录项的层级、标题和绝对字节 offset。
- 读取目录时必须校验源文件大小和修改时间；不一致就视为无有效目录。
- `BookmarkStore` 将书签保存到 `SharedPreferences` 的 `bookmarks` JSON 数组。书签包含 `id`、`bookId`、绝对字节 offset 和创建时间。
- 同一本书的同一 offset 只保存一个书签。删除书籍时同时删除对应目录、书签、阅读缓存和 TXT 本地副本。

## 3. 精确阅读位置

百分比只负责显示和滑杆跳转，不能承担精确恢复。当前两位小数显示不会损失阅读位置，因为存储与恢复使用绝对字节 offset。

保存位置流程：

1. 根据 `readerScroll` 的当前 Y 坐标找到正文布局中首个可见完整行；
2. 取得该行起始字符索引；
3. 通过当前 `CacheSegment` 的 `ByteOffsetMap` 把字符索引转换为窗口内字节位置；
4. 加上窗口起始 offset，得到源 TXT 的绝对 offset；
5. 保存 offset，并用 `offset / fileSize` 更新显示进度。

恢复位置执行反向映射：先找到覆盖目标 offset 的活动窗口，再用 `ByteOffsetMap.charIndexForByteOffset()` 定位字符和实际文本行。不得使用“字节比例等于页面高度比例”的近似算法。

UTF-8、UTF-16LE、UTF-16BE 等多字节编码必须在完整字符边界读取和切分；`ByteOffsetMap` 是字符位置与源文件字节位置之间的唯一映射层。

## 4. 阅读缓存

阅读器使用“连续组合 cache stack + 多页阅读滑动窗口 + 磁盘读取段 LRU + 持久化最近窗口”。磁盘访问只在初始化、显式跳转、后台补段和持久化读写时发生；屏幕显示始终只读取已经分页完成的阅读滑动窗口。

### 4.1 Cache segment

- 每段名义大小为 `128 KiB`。
- segment 从完整字符边界开始和结束，保存源文件起始 offset、实际字节数、解码文本和 `ByteOffsetMap`。
- 反向补段为了与当前窗口严格连续，遇到换行或字符边界对齐时允许略大于名义大小。
- 内存 segment LRU 最多保留 6 段；key 包含文件、文件大小、编码、起始 offset 和读取长度，避免错误复用。

### 4.2 Cache combine stack

`CacheCombineStack` 对阅读框架只暴露一个连续、不可变的 cache 快照；它不是 segment 容器，也不是 FIFO/Deque。segment 只作为后台磁盘读取和补充的输入，必须先与旧 cache 合并、裁剪，再为最终文本构建统一 `ByteOffsetMap`，阅读框架不得观察到 segment 边界。不可变快照及其 map 可安全共享，不能仅为后台副本重复构建。

- 单个磁盘 segment 名义大小为 `128 KiB`，组合窗口名义容量为两个 segment，即约 `256 KiB`，通常覆盖当前页前后很多页，而不是只预读下一页。
- 打开书籍或显式跳转时，以目标 offset 为中心读取前后两个连续 segment，并先合并成一个 cache 快照后再交给阅读框架。
- 每次普通翻页后都检查 cache stack 的 byte 水位；阅读位置距 stack 任一边缘不超过约 10% 时，后台读取对应方向的新 segment，执行“旧 cache + 新 segment -> 新连续 cache”，再从远端淘汰超出容量的内容。
- 磁盘读取、文本合并、统一字节映射重建和远端淘汰全部在后台执行；主线程不能在准备显示某一页时临时读盘或拼接 segment。
- 用户在读取期间改变方向时，后台结果必须再次校验书籍、加载请求和窗口代次；过期结果不能推动活动窗口。
- 新 cache 快照发布只替换后台 stack 引用，不调用 `TextView.setText()`、不修改 `scrollY`、不触发画面换绑，也不改变当前页面。

### 4.3 阅读页面滑动窗口

- `ReaderPageWindow` 保存前后多页已经排版完成、带绝对起止 byte offset 的不可变页面；`ReaderPageTextView` 直接裁剪绘制分页器生成的同一个 `StaticLayout`，不能再把单页字符串交给普通 `TextView` 二次排版。这样断词、连字符和双向文字都不会改变分页边界。
- 首次定位或排版参数变化时，从最新连续 cache 快照预排版当前页两侧各最多 8 页，形成最多 17 页的初始窗口。
- 此后正常每向前翻一页，就异步从 cache stack 排版一个新的下一页并 append 到窗口尾部；每向后翻一页则排版一个新的上一页并 prepend 到窗口头部。后台任务执行期间的多次快速翻页只累计缺口，下一任务把缺少的多页合并为一个批次补齐；任一侧低于 4 页时进入紧急补充，并优先恢复到 8 页。
- 页面窗口最多保留 17 页；每追加一页就淘汰相反方向离当前页最远的一页，当前页对象和当前 byte offset 不变。
- 普通翻页只把窗口的 current index 前移或后移，然后显示已经就绪的单页；它不读取磁盘、不合并 cache、不重新绑定整份 cache，也不依赖 segment 边界。
- cache stack 即使在后台合并和滑动，已进入页面窗口的页面仍由窗口独立持有，因此显示链路不存在 cache 换绑、中间布局帧或 segment 边界感，也不需要原子画面提交。
- 后台分页结果只有在书籍、不可变 cache 快照、排版代次和请求边界均仍匹配时才能进入窗口；新增页面内部及其与窗口边界之间的 byte offset 必须严格连续。旧批次、空洞和重叠结果全部丢弃并按当前状态重试。

### 4.4 持久化最近窗口与打开策略

最近阅读窗口延迟约 `800 ms` 在后台写入，不能在每次翻页时同步写盘。缓存包含源文件大小、窗口范围、解码文本和必要校验信息。

打开书籍时，最近窗口文件的读取、解码、校验和 `ByteOffsetMap` 构建全部在 I/O 执行器完成，主线程只接收已经准备好的不可变结果：

1. 如果有效缓存覆盖保存的目标 offset，直接在该窗口内精确恢复，不再按同一 offset 重复初始化窗口；
2. 如果缓存不覆盖目标 offset，或源文件大小、修改时间不匹配，则不展示错误位置的缓存预览，直接在后台读取覆盖目标的双段窗口；
3. 两条路径都只向页面滑动窗口提交一次正确位置；覆盖判断不会停止接近边缘时的后台相邻段续读。

## 5. 大文件索引、目录与搜索

- 阅读定位索引按约 `64 KiB` 间隔在后台构建可读字节边界，帮助大文件进度跳转；它不替代最终的字符/字节精确映射。
- `TocGenerator` 流式逐行扫描 TXT，不把整本书载入内存。它识别卷、章、节标题，按文件实际存在的标题类型把层级归一化到一至三级，并记录标题行起始绝对 offset。
- 目录自动生成由全局开关控制；手工生成不依赖该开关。生成过程使用单线程后台执行器，写入成功后才刷新目录页面。
- 搜索使用 `64 KiB` 字节块流式扫描，每批最多 200 条。搜索结果 offset 只用于临时阅读；返回搜索时恢复 `SearchSession` 保存的正式阅读位置。

## 6. 并发与生命周期

- 正文窗口读取（包括持久缓存恢复）、阅读缓存写入、搜索和目录生成使用相互独立的单线程执行器，避免同类任务乱序。
- 磁盘补段与页面预分页使用独立单线程执行器和独立请求代次。segment 合并结果先形成不可变 cache 快照；页面结果形成不可变 page 列表，两者都不能原地修改前端当前页。
- `onPause()` 停止自动翻页、撤销阅读时亮屏、保存正式进度并刷新待写缓存。
- 自动翻页每次触发时只在阅读框架空闲时调用一次真实 `pageForward()`，随后按已保存的间隔重新安排；动画、窗口加载或进度冻结期间不强行翻页。
- 从阅读页进入设置会停止自动翻页并撤销亮屏；返回时重新创建阅读框架，若上下菜单原本可见则直接恢复可见状态，不再播放一次菜单滑入动画。
- `onDestroy()` 移除 Handler 回调并关闭全部执行器。
- 自动翻页只有在 Activity 前台、正式阅读视图就绪、无搜索/临时阅读、无动画、无窗口加载且未冻结进度写入时才允许执行。

## 7. 构建边界

- `version.properties` 是 `versionCode` 和 `versionName` 的唯一来源。
- 成功完成 `assembleDebug` 或 `assembleRelease` 后，Gradle 的 `bumpVersion` 会把 `versionCode` 增加 1。
- 普通代码审查、文档整理和静态检查不得触发构建；只有用户明确要求时才执行 Debug/Release 编译或 APK 打包。
