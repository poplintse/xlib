# Xlib 架构

## 阅读缓存

阅读器使用两层内存模型，展示路径不直接访问磁盘。

### Cache segment

- 每段名义大小为 `128 KiB`。
- segment 必须从完整字符边界开始，并在完整字符边界结束。
- segment 保存源文件起始字节、实际字节数、解码文本和字符/字节映射。
- 反向补段为了和当前窗口严格连续，遇到换行对齐时允许略大于名义大小。

选择 `128 KiB` 的原因：本地顺序读取延迟低；两个 segment 的基础窗口只有约
`256 KiB` 源数据；10% 预取区约 `25.6 KiB`，通常能覆盖多页文本，足以隐藏后台 I/O。

### Cache combine stack

`CacheCombineStack` 实际使用定长 `Deque`，表示由两个 segment 组成的双端移动窗口，
而不是两个严格 FIFO 队列。

- 打开书籍或跳转位置时，以目标 offset 为中心一次加载两个连续 segment。
- 向后阅读至窗口右侧剩余不超过 10% 时，后台读取右侧相邻 segment。
- 右侧 segment 就绪后追加到窗口，并淘汰最左 segment。
- 向前阅读时执行镜像操作：前插左侧 segment，并淘汰最右 segment。
- 用户在读取期间改变方向时，后台结果会重新检查当前位置；不再需要的结果只保留在
  LRU segment cache 中，不推动活动窗口。
- 换绑组合文本、恢复 offset 和解除进度写入冻结属于一个原子 UI 过程。

### I/O 约束

- 磁盘读取只发生在初始定位、显式进度跳转和 10% 阈值触发的后台补段。
- 普通前后翻页只读取活动 combine stack，不在页边界临时访问磁盘。
- 补段任务带书籍、加载请求和窗口代次校验，过期任务不能覆盖当前窗口。
- segment LRU key 包含文件、文件大小、编码、起始 offset 和读取长度，避免不同长度的
  segment 错误复用。
